Ubuntu server

## View running processes
```bash
ps aux
```
shows every running process (like task manager in windows)

## Narrow down processes by adding grep
```bash
ps aux | grep ssh
ps aux | grep michael
```
lets you search for specific processes by command/user etc

## Real-Time Monitoring
Installing htop for monitoring
```bash
sudo apt install htop
```

running:
```bash
htop
```
Reviewing:
- cpu usage
- ram usage
- running processes

purpose:
view running processes and system resoruce usage.

## Systemd
similar to windows services
View running services:
```bash
systemctl list-units --type=service
```

## Check specific service
```bash
systemctl status SERVICE
```
Allows to review the status of a specific service and more details about the service.

## Service Commands
Check status
```bash
systemctl status SERVICE
```

Start:
```bash
sudo systemctl start SERVICE
```

Stop:
```bash
sudo systemctl stop SERVICE
```

Restart:
```bash
sudo systemctl restart SERVICE
```

Enable at boot:
```bash
sudo systemctl enable SERVICE
```

## Investigating logs
Access Logs:
```bash

```

Error log:
```bash

```

Tested generating traffic by refreshing webpage in browser then reviewing logs again.

## Real-time monitoring
```bash
sudo tail -f /var/log/nginx/access.log
```
Tested to watch how logs update as more traffic is generated by refreshing the page again

## journalctl
System logs
```bash
journalctl
```

Recent logs:
```bash
journalctl -n 50
```

Logs for SSH:
```bash
journalctl -u ssh
```

Logs for nginx:
```bash
journalctl -u nginx
```

Only recent logs for nginx
```bash
journalctl -u nginx -n 20
```
